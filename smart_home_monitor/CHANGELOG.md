## 1.1.0-beta3 — Critical bugfixes

### Исправлено
- 🔒 **`db_root_password`** — тип поля изменён с `str?` на `password?`: значение теперь скрыто в HA UI и не попадает в логи (как остальные пароли в конфиге)
- 🔢 **Версия аддона** — убрана хардкоженная `"1.0.0"` в `main.py` и `/api/health`; теперь читается из `config.json` в рантайме — единственный источник правды
- 🔋 **Фильтр низкой батареи** (`GET /api/batteries/?low_only=true`) — исправлен: использовал устаревшее поле `telegram_battery_threshold` вместо `_battery_threshold` с правильной fallback-цепочкой
- 📨 **MAX мессенджер** — исправлена логика вызова: MAX вызывался в `finally`-блоке и срабатывал даже при ошибке Telegram; теперь оба канала работают независимо

## 1.1.0-beta2 — Auto DB provisioning

### Новое
- 🗄️ **Автосоздание базы данных** при первом запуске аддона — если указан `db_root_password`, аддон сам создаст базу, пользователя и выдаст права
- Логика идемпотентна: при повторном запуске проверяет доступность БД и пропускает provisioning если всё уже настроено
- При ошибке (root недоступен, нет пароля) — выводит в лог точные SQL-команды для ручного создания и продолжает запуск

### Конфигурация
Новый необязательный параметр `db_root_password` — нужен только для первоначального создания БД:
```yaml
db_root_password: "your_mariadb_root_password"
```
Если параметр не указан и БД уже существует — аддон работает как прежде.

## 1.1.0-beta1 — No-flicker refresh

### Исправлено
- 🔄 **Устранён сброс страницы при автообновлении** — вместо полного пересоздания DOM теперь применяется diff-патч: обновляются только изменившиеся ячейки (статус, батарея, LQI)
- 🔒 **Guard при открытой модалке** — пока открыто окно устройства/батареи/графика, перерисовка списков полностью заблокирована; данные обновляются в фоне
- 📋 **Списки алертов и энергопотребления** — получили аналогичный diff-патч (по id уведомления и ключу метрики)

## 1.1.0 — Первый стабильный релиз

### Основные возможности
- 📡 Мониторинг Zigbee устройств — статус онлайн/офлайн, LQI, батарея
- 🔋 Заряд батарей — график разряда, пагинация
- ⚡ Токопотребление — мощность, ток, напряжение, энергия с графиком
- 📈 Конструктор графиков — любые метрики, период от 3 часов до 30 дней
- 🔔 История уведомлений — пагинация, автоудаление через 24ч
- 📨 Telegram — алерты офлайн, батареи, превышения порогов, ежедневный отчёт
- 🖼 Картинки устройств из Zigbee2MQTT или свои

### Настройки
- Таймаут офлайн по умолчанию: 180 минут (глобально и для каждого устройства отдельно)
- Пороги МИН/МАКС для числовых метрик с TG-уведомлением
- Кастомный текст Telegram сообщений для каждого порога
